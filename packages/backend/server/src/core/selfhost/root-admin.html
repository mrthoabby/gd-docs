Autenticación

        
Configuración de registro de usuarios y OAuth

      

      

        
Opciones de registro

        

          

            
Permitir registro público
auth.allowSignup — Si está OFF, solo admins pueden crear cuentas

            
          

          

            
Verificación de email requerida
auth.requireEmailVerification — Recomendado OFF en self-hosted

            
          

          

            
Permitir OAuth para nuevos usuarios
auth.allowSignupForOauth — Login con Google/GitHub para cuentas nuevas

            
          

        

        

          
        

      


      
      

        
🔒 Restricción de registro por dominio / email

        

          Si dejás ambos campos vacíos, cualquier email puede registrarse (comportamiento por defecto).

          Si configurás al menos uno, solo los emails que coincidan con la lista podrán crear cuenta nueva.

          No afecta a usuarios que ya tienen cuenta. Aplica a email/password y a OAuth por igual.
        


        

          
          
          
Ej: mercadolibre.com.co → solo emails @mercadolibre.com.co podrán registrarse

        


        

          
          
          
Emails fuera de los dominios permitidos que igual pueden registrarse (p.ej. proveedores externos)

Co-edición colaborativa

        

          ✅ La co-edición funciona 100% con tu base de datos propia.
          El servidor usa YJS (CRDT) sobre WebSocket (Socket.IO), sincronizando con PostgreSQL.
          No hay dependencia de servidores externos.
        

        

          
            
ProtocoloYJS CRDT sobre Socket.IO WebSocket

            
PersistenciaPostgreSQL (tablas: Snapshot, Update)

            
Awareness (cursores)Tiempo real, en memoria

            
FallbackHTTP polling automático si WS falla

Comandos Docker

        

          

            
VER LOGS

            
docker compose -f .docker/selfhost/compose.yml logs -f gddocs

            
docker compose -f .docker/selfhost/compose.yml logs --tail=100 gddocs

          

          

            
GESTIÓN

            
docker compose -f .docker/selfhost/compose.yml ps

            
docker compose -f .docker/selfhost/compose.yml restart gddocs

          

          

            
REPARAR SIN PERDER DATOS

            
bash .docker/selfhost/repair.sh

            
bash .docker/selfhost/repair.sh --status

          

          

            
ACTUALIZAR

            
bash .docker/selfhost/update.sh

            
bash .docker/selfhost/repair.sh --rebuild

          

        

        
💡 Click en cualquier comando para copiarlo al portapapeles.
